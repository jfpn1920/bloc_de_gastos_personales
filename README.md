# 👋 ¡Bienvenidos! a mi proyecto Bloc de Gastos Personales
<img src="imagen_presentacion.png" alt="Presentación" width="220" align="left" style="margin-right: 20px; border-radius: 5px;">
<p style="text-align: justify;">
    ¡Bienvenido a este repositorio! Aquí encontrarás la presentación de mi proyecto <b>bloc de gastos personales</b>, una propuesta enfocada en el registro, control y administración de gastos personales mediante diferentes tecnologías y enfoques de programación. Este proyecto fue desarrollado como ejercicio de aprendizaje utilizando <b>pseint, python, java, visual basic, frontend y backend</b>, permitiendo fortalecer conocimientos en lógica de programación, estructuras de datos, desarrollo de software y manejo de bases de datos.
</p>
<p style="text-align: justify;">
    A lo largo del proyecto se implementaron diferentes versiones del mismo sistema, comenzando desde aplicaciones de consola hasta una aplicación web completa con persistencia de datos en <b>localstorage</b> y almacenamiento en base de datos utilizando <b>xampp y phpmyadmin</b>. De esta manera, el repositorio se convierte en una práctica integral orientada al fortalecimiento de habilidades en programación, desarrollo frontend, backend e integración de tecnologías.
</p>
<br clear="left"/>

#
### ✨ Características del proyecto
* Gestión organizada de gastos personales.
* Desarrollo progresivo del mismo sistema en múltiples lenguajes de programación.
* Aplicaciones de consola e interfaz web funcional.
* Almacenamiento local utilizando <b>localstorage</b>.
* Integración completa entre <b>frontend y backend</b>.
* Implementación de base de datos con <b>xampp y phpmyadmin</b>.
* Operaciones crud funcionales (crear, leer, actualizar y eliminar gastos).
* Actualización dinámica de registros en tiempo real.
* Sistema de control de presupuesto mensual.
* Cálculo automático de:
    * Total gastado
    * Presupuesto disponible
    * Exceso de gasto
* Visualización detallada de gastos registrados.
* Vista previa de cada gasto.
* Edición y eliminación dinámica de registros.
* Tablero interactivo de gastos.
* Manipulación dinámica del DOM con JavaScript.
* Comunicación entre frontend y backend mediante <b>fetch api</b>.
* Manejo de datos en formato <b>json</b>.
* Proyecto desarrollado sin frameworks.
* Organización estructurada de archivos y código.
* Enfoque orientado al aprendizaje progresivo de programación.
